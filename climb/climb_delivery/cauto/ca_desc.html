A cellular automaton is a collection of cells on a grid of specified shape that evolves through a number of discrete time steps according to a set of rules based on the states of neighboring cells. Check out Wikipedia for more information.

In this problem we will simulate a circular one-dimensional cellular automata with Rule 30. Circular in the sense that the first element is neighboring the last. Rule 30 looks like this:

current pattern 111 110 101 100 011 010 001 000
new state for center cell 0 0 0 1 1 1 1 0

Input

The input consists of an initial state, and the number of generations to be simulated.
00100
2

Ouput

The end state.
11001
